## Day 89: Advanced Git Configurations & Best Practices
Today, I explored essential best practices and advanced configuration commands to optimize repository management before starting a new project.
### Key Learnings
 * **The .gitignore File**:
   * Prevents tracking large, unnecessary, or sensitive files (like virtual environments venv/, local configurations, data folders, or credentials).
   * Helps optimize remote storage space and prevents cloud push rejection due to massive file uploads.
 * **Global Configurations**:
   * Set up identity properties globally so every commit is correctly attributed.
 * **Essential Setup Commands**:
   * git config --global user.name "Your Name": Configures your global username commit property.
   * git config --global user.email "youremail@example.com": Sets up the global email property linked to your GitHub account.
   * touch .gitignore: Creates the tracking exclusion file in the root of your local repository.
