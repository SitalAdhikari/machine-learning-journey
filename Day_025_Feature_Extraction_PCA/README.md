## Day 25 – Feature Extraction using PCA

Today I learned about **Principal Component Analysis (PCA)**, a powerful technique for **feature extraction and dimensionality reduction**.

---

### What is PCA?

PCA transforms the original features into a new set of features called **Principal Components**, which:

* Are **uncorrelated**
* Capture the **maximum variance** in the data
* Reduce the number of dimensions while preserving important information

---

### Why Use PCA?

* Reduces **dimensionality** of data
* Helps in **visualization** (2D/3D)
* Removes **multicollinearity**
* Improves performance of some models

---

### What I Learned

* Importance of **feature scaling before PCA**
* How PCA projects data onto new axes (principal components)
* Selecting number of components based on **explained variance**
* Trade-off between **information loss and dimensionality reduction**

---

### Key Insight

PCA does not just reduce features — it creates **new meaningful features** that capture the most important patterns in the data.

---

### Limitation

* Reduced interpretability (new features are combinations of original ones)
* Works best with **linear relationships**

---

### Final Takeaway

PCA is a powerful tool when dealing with **high-dimensional data**, helping simplify datasets while retaining essential information.
